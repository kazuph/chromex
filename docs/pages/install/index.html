Chromex 설치 가이드

Chrome Web Store에서 Chromex를 설치했다면, 한 번만 로컬 브리지를 연결해야 합니다. 아래 순서대로 진행하면 소스코드 빌드 없이 사용할 수 있습니다.

10분 내외 Windows / macOS 복사해서 붙여넣기 중심
왜 추가 설치가 필요한가요? Chrome 확장 프로그램은 보안상 내 컴퓨터 프로그램을 직접 설치할 수 없습니다. 그래서 사용자가 로컬 브리지를 한 번 등록해야 합니다.
소스코드 빌드는 필요 없습니다 스토어 사용자는 개발자용 빌드 과정을 하지 않습니다. 필요한 것은 Node.js, Codex CLI, Chromex Local Bridge ZIP입니다.
ChatGPT 계정으로 사용 가능 로컬 브리지가 연결되면 Chromex에서 ChatGPT 계정 로그인 또는 API 키 로그인을 선택할 수 있습니다.
1

Node.js LTS 설치

먼저 공식 Node.js 페이지에서 LTS 버전을 설치합니다. Windows는 Windows Installer, macOS는 macOS Installer를 받으면 됩니다.

Node.js 공식 다운로드 열기

설치 화면에서는 특별히 바꿀 것 없이 Next 또는 계속을 눌러 완료하면 됩니다. 설치가 끝나면 열려 있던 터미널을 닫고 새로 여세요.
2

Codex CLI 설치

Windows는 PowerShell, macOS는 터미널을 열고 아래 명령어를 복사해서 붙여넣습니다.

npm install -g @openai/codex

설치가 끝나면 아래 명령어로 확인합니다.

codex --version
codex-cli와 버전 숫자가 보이면 성공입니다.

OpenAI Codex CLI 공식 안내 열기

3

Chromex Local Bridge ZIP 다운로드

아래 파일을 내려받고 압축을 풉니다. 보통 다운로드 폴더에 저장됩니다.

chromex-local-bridge.zip 다운로드

압축을 푼 폴더 안에 scripts, bridge, native-host 폴더가 보여야 합니다.
4

로컬 브리지 등록

Windows

압축을 푼 chromex-extension 폴더를 열고, 빈 공간에서 오른쪽 클릭 후 터미널에서 열기 또는 PowerShell에서 열기를 누릅니다. 그 다음 아래 명령어를 실행합니다.

node scripts/install-native-host.mjs --browser=chrome

macOS

터미널을 열고 cd 를 입력한 뒤, 압축을 푼 chromex-extension 폴더를 터미널 창에 끌어다 놓고 Enter를 누릅니다. 그 다음 같은 명령어를 실행합니다.

스토어 확장 ID는 설치 파일에 자동으로 포함되어 있어서 일반적으로 따로 입력하지 않아도 됩니다.
5

Chrome 완전히 껐다 켜기

모든 Chrome 창을 완전히 닫고 다시 엽니다. Chromex 사이드패널에서 연결 확인을 누르면 됩니다.

로컬 브리지가 연결되면 ChatGPT 계정으로 계속 또는 API 키로 사용 버튼을 사용할 수 있습니다.

막히는 경우

  • node 또는 npm을 찾을 수 없다고 나오면 Node.js 설치 후 터미널을 완전히 닫고 새로 여세요.
  • codex --version이 안 되면 Codex CLI 설치가 끝나지 않은 상태입니다. 2단계 명령어를 다시 실행하세요.
  • Windows에서 Codex 실행 파일을 못 찾으면 Chromex 설정에서 Codex 경로를 %APPDATA%\npm\codex.cmd로 지정한 뒤 Chrome을 다시 여세요.
  • 계속 로컬 브리지를 기다린다고 나오면 Chrome을 완전히 종료한 뒤 다시 열고, Chromex에서 연결 확인을 누르세요.
  • 회사 PC, VPN, 보안 프로그램 환경에서는 설치 파일 실행이나 네트워크 연결이 막힐 수 있습니다. 이 경우 사내 보안 정책 확인이 필요합니다.

Chromex는 Chrome 보안 정책을 우회하지 않습니다. 이 가이드는 Chrome Web Store 설치자가 필요한 로컬 브리지를 직접 등록하도록 돕기 위한 공개 안내입니다.
