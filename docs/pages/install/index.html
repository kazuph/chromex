Chromex Store Install Guide

Choose your language and follow the one-time local bridge setup. Chrome Web Store users do not need to build Chromex from source.

English 한국어 日本語 简体中文

English

Install Node.js, install the official Codex CLI, download the Chromex Local Bridge ZIP, register it once, then restart Chrome.

About 10 minutes Windows / macOS Copy and paste friendly
Why is this needed?Chrome extensions cannot install local programs by themselves. The local bridge is registered by the user so Chrome can safely start Codex through Native Messaging.
No source build requiredStore users only need Node.js, Codex CLI, and the Chromex Local Bridge ZIP.
ChatGPT login is supportedAfter the bridge connects, use ChatGPT login or API key login inside Chromex.
1

Install Node.js LTS

Open the official Node.js page and install the LTS version. Use the Windows Installer on Windows or the macOS Installer on macOS.

Open Node.js download

You can keep the default installer choices. After installation, close your terminal and open a new one.
2

Install Codex CLI

Open PowerShell on Windows or Terminal on macOS, then run:

npm install -g @openai/codex

Check that it works:

codex --version
If you see codex-cli and a version number, this step worked.

Open official Codex CLI guide

3

Download Chromex Local Bridge

Download and unzip the bridge package. It usually lands in your Downloads folder.

Download chromex-local-bridge.zip

After unzipping, the folder should contain scripts, bridge, and native-host.
4

Register the local bridge

Windows

Open the unzipped chromex-extension folder, right-click empty space, choose Open in Terminal or Open in PowerShell, then run:

node scripts/install-native-host.mjs --browser=chrome

macOS

Open Terminal, type cd , drag the unzipped chromex-extension folder into Terminal, press Enter, then run the same command above.

The Chrome Web Store extension ID is already allowed by the bridge package, so most users do not need to type an extension ID.
5

Restart Chrome completely

Quit every Chrome window, open Chrome again, then press Check connection in Chromex.

When the bridge connects, the Continue with ChatGPT and Use API key buttons become available.

Troubleshooting

  • If node or npm is not found, reinstall Node.js and open a new terminal.
  • If codex --version fails, run the Codex CLI install command again.
  • On Windows, if Codex is not detected, set the optional Codex binary path in Chromex to %APPDATA%\npm\codex.cmd, save, then restart Chrome.
  • If Chromex still waits for the local bridge, fully quit Chrome and reopen it before pressing Check connection.

Chromex 설치 가이드

Node.js 설치, 공식 Codex CLI 설치, Chromex Local Bridge ZIP 다운로드, 로컬 브리지 등록, Chrome 재시작 순서로 진행하세요.

10분 내외 Windows / macOS 복사해서 붙여넣기 중심
왜 추가 설치가 필요한가요?Chrome 확장 프로그램은 보안상 내 컴퓨터 프로그램을 직접 설치할 수 없습니다. 사용자가 로컬 브리지를 한 번 등록해야 Chrome이 Codex를 안전하게 시작할 수 있습니다.
소스코드 빌드는 필요 없습니다스토어 사용자는 Node.js, Codex CLI, Chromex Local Bridge ZIP만 준비하면 됩니다.
ChatGPT 계정으로 사용 가능로컬 브리지가 연결되면 Chromex에서 ChatGPT 계정 로그인 또는 API 키 로그인을 선택할 수 있습니다.
1

Node.js LTS 설치

공식 Node.js 페이지에서 LTS 버전을 설치합니다. Windows는 Windows Installer, macOS는 macOS Installer를 받으면 됩니다.

Node.js 공식 다운로드 열기

설치 화면에서는 특별히 바꿀 것 없이 Next 또는 계속을 눌러 완료하면 됩니다. 설치가 끝나면 열려 있던 터미널을 닫고 새로 여세요.
2

Codex CLI 설치

Windows는 PowerShell, macOS는 터미널을 열고 아래 명령어를 복사해서 붙여넣습니다.

npm install -g @openai/codex

설치가 끝나면 아래 명령어로 확인합니다.

codex --version
codex-cli와 버전 숫자가 보이면 성공입니다.

OpenAI Codex CLI 공식 안내 열기

3

Chromex Local Bridge ZIP 다운로드

아래 파일을 내려받고 압축을 풉니다. 보통 다운로드 폴더에 저장됩니다.

chromex-local-bridge.zip 다운로드

압축을 푼 폴더 안에 scripts, bridge, native-host 폴더가 보여야 합니다.
4

로컬 브리지 등록

Windows

압축을 푼 chromex-extension 폴더를 열고, 빈 공간에서 오른쪽 클릭 후 터미널에서 열기 또는 PowerShell에서 열기를 누릅니다. 그 다음 아래 명령어를 실행합니다.

node scripts/install-native-host.mjs --browser=chrome

macOS

터미널을 열고 cd 를 입력한 뒤, 압축을 푼 chromex-extension 폴더를 터미널 창에 끌어다 놓고 Enter를 누릅니다. 그 다음 같은 명령어를 실행합니다.

스토어 확장 ID는 설치 파일에 자동으로 포함되어 있어서 일반적으로 따로 입력하지 않아도 됩니다.
5

Chrome 완전히 껐다 켜기

모든 Chrome 창을 완전히 닫고 다시 엽니다. Chromex 사이드패널에서 연결 확인을 누르면 됩니다.

로컬 브리지가 연결되면 ChatGPT 계정으로 계속 또는 API 키로 사용 버튼을 사용할 수 있습니다.

막히는 경우

  • node 또는 npm을 찾을 수 없다고 나오면 Node.js 설치 후 터미널을 완전히 닫고 새로 여세요.
  • codex --version이 안 되면 Codex CLI 설치가 끝나지 않은 상태입니다. 2단계 명령어를 다시 실행하세요.
  • Windows에서 Codex 실행 파일을 못 찾으면 Chromex 설정에서 Codex 경로를 %APPDATA%\npm\codex.cmd로 지정한 뒤 Chrome을 다시 여세요.
  • 계속 로컬 브리지를 기다린다고 나오면 Chrome을 완전히 종료한 뒤 다시 열고, Chromex에서 연결 확인을 누르세요.

日本語

Node.js、公式 Codex CLI、Chromex Local Bridge ZIP を準備し、ローカルブリッジを一度だけ登録して Chrome を再起動します。

約 10 分Windows / macOSコピー＆ペースト中心
なぜ必要ですか?Chrome 拡張機能はローカルプログラムを自動インストールできません。ユーザーがブリッジを登録することで、Chrome が Native Messaging 経由で Codex を安全に起動できます。
ソースビルドは不要ストア版のユーザーに必要なのは Node.js、Codex CLI、Chromex Local Bridge ZIP だけです。
ChatGPT ログイン対応ブリッジ接続後、Chromex で ChatGPT ログインまたは API キーログインを選択できます。
1

Node.js LTS をインストール

公式 Node.js ページから LTS をインストールしてください。Windows は Windows Installer、macOS は macOS Installer を選びます。

Node.js ダウンロードを開く

インストーラーは基本的に既定値のままで進めて問題ありません。完了後、ターミナルを閉じて開き直してください。
2

Codex CLI をインストール

Windows では PowerShell、macOS では Terminal を開いて実行します。

npm install -g @openai/codex

動作確認:

codex --version
codex-cli とバージョン番号が表示されれば成功です。

公式 Codex CLI ガイドを開く

3

Chromex Local Bridge ZIP をダウンロード

以下のファイルをダウンロードして展開します。通常は Downloads フォルダに保存されます。

chromex-local-bridge.zip をダウンロード

展開後、scripts、bridge、native-host フォルダがあることを確認してください。
4

ローカルブリッジを登録

Windows

展開した chromex-extension フォルダを開き、空白部分を右クリックして Open in Terminal または Open in PowerShell を選び、次を実行します。

node scripts/install-native-host.mjs --browser=chrome

macOS

Terminal で cd と入力し、展開した chromex-extension フォルダを Terminal にドラッグして Enter を押します。その後、同じコマンドを実行します。

ストア版の拡張 ID はブリッジパッケージで許可済みのため、通常は ID を入力する必要はありません。
5

Chrome を完全に再起動

すべての Chrome ウィンドウを閉じて開き直し、Chromex の Check connection を押します。

接続できると、Continue with ChatGPT または Use API key が使えるようになります。

困ったとき

  • node や npm が見つからない場合は、Node.js をインストールし直して新しいターミナルを開いてください。
  • codex --version が失敗する場合は、Codex CLI のインストールをもう一度実行してください。
  • Windows で Codex が検出されない場合は、Chromex の optional Codex binary path に %APPDATA%\npm\codex.cmd を設定して Chrome を再起動してください。

简体中文

安装 Node.js、安装官方 Codex CLI、下载 Chromex Local Bridge ZIP、注册一次本地 bridge，然后重启 Chrome。

约 10 分钟Windows / macOS适合复制粘贴
为什么需要这一步?Chrome 扩展出于安全限制不能自行安装本地程序。用户注册本地 bridge 后，Chrome 才能通过 Native Messaging 安全启动 Codex。
不需要源码构建商店版用户只需要 Node.js、Codex CLI 和 Chromex Local Bridge ZIP。
支持 ChatGPT 登录bridge 连接后，可以在 Chromex 中选择 ChatGPT 登录或 API key 登录。
1

安装 Node.js LTS

打开 Node.js 官方页面并安装 LTS 版本。Windows 选择 Windows Installer，macOS 选择 macOS Installer。

打开 Node.js 下载页

安装时保持默认选项即可。完成后关闭终端，再打开一个新的终端。
2

安装 Codex CLI

Windows 打开 PowerShell，macOS 打开 Terminal，然后运行:

npm install -g @openai/codex

确认安装成功:

codex --version
如果看到 codex-cli 和版本号，说明成功。

打开官方 Codex CLI 指南

3

下载 Chromex Local Bridge ZIP

下载并解压下面的文件。通常会保存在 Downloads 文件夹。

下载 chromex-local-bridge.zip

解压后应能看到 scripts、bridge、native-host 文件夹。
4

注册本地 bridge

Windows

打开解压后的 chromex-extension 文件夹，在空白处右键，选择 Open in Terminal 或 Open in PowerShell，然后运行:

node scripts/install-native-host.mjs --browser=chrome

macOS

打开 Terminal，输入 cd ，把解压后的 chromex-extension 文件夹拖入 Terminal，按 Enter，然后运行同一个命令。

商店版扩展 ID 已在 bridge 包中允许，大多数用户不需要手动输入扩展 ID。
5

完全重启 Chrome

关闭所有 Chrome 窗口并重新打开，然后在 Chromex 中点击 Check connection。

连接成功后，Continue with ChatGPT 和 Use API key 按钮会可用。

排查问题

  • 如果找不到 node 或 npm，请重新安装 Node.js，并打开新的终端。
  • 如果 codex --version 失败，请重新运行 Codex CLI 安装命令。
  • Windows 上如果检测不到 Codex，请在 Chromex 设置中把 optional Codex binary path 设置为 %APPDATA%\npm\codex.cmd，保存后重启 Chrome。

Chromex does not bypass Chrome security policies. This public guide helps Chrome Web Store users register the local bridge that Chrome requires for native messaging.
